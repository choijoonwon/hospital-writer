# 현장 방문 전 확인 질문 리스트

---

## 🗂 스프레드시트 구조

1. **스프레드시트가 Google Sheets인지, Excel인지?**
   - Excel이면 연동 방식이 달라짐

2. **컬럼 순서가 어떻게 되어 있는지?**
   - 지금 만든 것과 순서 다르면 `backend/sheets.py`의 `COLUMNS` 배열 수정 필요

3. **지금 병원명 컬럼이 따로 있는지?**
   - 없으면 스프레드시트에 B열 추가 필요

4. **시트 탭이 여러 개인지? 병원마다 탭이 따로 나뉘어 있는지?**
   - 예: 탭1=A성형외과, 탭2=B성형외과 형태면 코드 조정 필요

5. **한 시트에 환자가 몇 명 정도 있는지?**
   - 규모 파악용

6. **언급 금지 항목, 특이사항 컬럼이 이미 있는지?**

---

## 🏥 병원 & 업무 방식

7. **현재 담당하는 병원이 몇 곳인지?**
   - 필터 버튼 구성 확인

8. **생성된 글 초안을 환자에게 어떻게 전달하는지?**
   - 카카오톡? 문자? 이메일? → 나중에 전달 기능 추가 여부 결정

9. **한 환자당 글을 몇 개 정도 만드는지?**
   - 1개인지, 플랫폼마다 각각인지

10. **현재 글 쓸 때 참고하는 예시 글이 있는지?**
    - 있으면 프롬프트에 반영해서 품질 높일 수 있음

---

## 💻 PC 환경

11. **Windows 버전이 몇인지?**
    - Windows 10 / 11 확인

12. **Python이 이미 설치되어 있는지?**

13. **인터넷이 되는지?**
    - 패키지 설치, API 호출 모두 인터넷 필요

14. **이 프로그램 쓸 PC가 몇 대인지?**
    - 여러 대면 각각 세팅하거나 내부 서버로 공유하는 방식 고려

---

## 🔑 API & Google 계정

15. **Google 계정이 있는지?**
    - Google Cloud 서비스 계정 만들 때 필요

16. **스프레드시트 소유자가 누구인지?**
    - 공유 권한 설정할 때 필요

17. **Claude API 또는 OpenAI API 계정이 있는지?**
    - 없으면 현장에서 같이 만들어야 함 (카드 결제 필요)

18. **API 비용은 누가 부담하는지?**
    - 월 사용량에 따라 비용 발생

---

## ✍️ 글 품질 관련

19. **지금 직원이 글 쓸 때 참고하는 가이드나 샘플이 있는지?**
    - 있으면 프롬프트에 바로 반영 가능

20. **플랫폼마다 금지어나 특별히 피해야 할 표현이 있는지?**

21. **생성된 글을 직접 수정해서 쓰는 편인지, 그대로 쓰는 편인지?**
    - 수정이 많다면 추가 기능 고려

22. **글 길이 기준이 있는지?**
    - 예: 여우야 게시글은 무조건 300자 이상 등

---

> **현장 세팅 순서**: 스프레드시트 컬럼 확인 → Google Cloud 세팅 → API 키 발급 → 테스트
> 이 순서로 진행하면 30~40분 안에 완료 가능합니다.
